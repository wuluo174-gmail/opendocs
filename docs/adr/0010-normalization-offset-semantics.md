# ADR-0010: 标准化后偏移量对应标准化文本而非原始文件

- 状态：已接受（已知限制）
- 日期：2026-03-08
- 关联阶段：S2

## 上下文

`ParserRegistry.parse()` 在解析后对段落文本执行标准化（NFC、全角→半角、空白清理），然后用标准化后的段落重新拼接 `raw_text` 并重新计算偏移量。这意味着 `ParsedDocument.raw_text` 及其 `Paragraph.start_char` / `end_char` 对应的是标准化后的文本，而非原始文件内容。

## 决策

S2 阶段接受此行为：

1. **内部一致性已验证**：`raw_text[start_char:end_char] == paragraph.text` 始终成立（`TestNormalizationOffsetIntegrity` 已覆盖）。
2. **标准化后的文本对 embedding 和检索更有利**：统一字符形式提升向量相似度和全文检索的召回率。
3. **不保留原始偏移映射**：MVP 阶段无此需求，避免过度设计。

## 影响

- 当前阶段（S2–S4）内部流程无影响：chunk → embedding → 检索均基于标准化文本。
- **FR-015（证据检查界面）**：如果需要"点击引用 → 跳转到原文件精确位置"，标准化可能导致偏移偏差（例如 NFC 合并组合字符后长度变化）。届时需要：
  - 在 `ParsedDocument` 中额外保存 `original_raw_text`，或
  - 在标准化时记录字符级映射表。
- 对于全角→半角转换，字符数量不变（1:1 映射），不影响偏移。
- 对于 NFC 合并（如 `e` + `◌́` → `é`），长度会缩短，偏移会偏移。

## 回退方案

在 FR-015 实施阶段，根据实际需求决定是否引入原始偏移映射。当前阶段的设计不阻塞后续扩展。
