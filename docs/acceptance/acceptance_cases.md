# OpenDocs 可执行验收用例 v1.0

> 本文件用于把《OpenDocs 工程实施主规范 v1.0》的验收矩阵落成可直接执行的测试规约。  
> 它既是人工验收手册，也是 `scripts/run_acceptance.py` 的用例蓝本。  
> 若与主规范冲突，以主规范为准；若与 `tasks.yaml` 冲突，以本文件中的用例通过标准为准。

---

## 0. 验收目标

验收的目标不是证明“系统大致可用”，而是严格验证以下红线是否成立：

1. **证据优先**：事实性回答默认带引用，且引用可打开、可核验。
2. **本地优先**：`Local-Only` 模式无外网请求。
3. **写操作受控**：`move / rename / create` 必须先预览、再确认、再执行，并有审计与回滚。
4. **记忆不越权**：记忆不能覆盖文档证据；冲突时以文档证据为准。
5. **可审计可恢复**：关键行为可查询，最近一次批量写操作可回滚。
6. **可交付**：一键验收脚本、性能测试、打包与用户文档齐备。

---

## 1. 执行原则

### 1.1 通过判定

以下条件同时满足，整体验收才算通过：

- TC-001 到 TC-019 全部通过
- TC-020 中的 **Blocker 子项** 全部通过
- 主规范中的三条红线全部通过：
  - Local-Only 网络为 0
  - 所有物理写操作均有预览/确认/审计/回滚
  - 事实问答默认附引用且证据不足拒答
- 无 Blocker 级缺陷
- 自动化报告与人工抽查结论一致

### 1.2 失败分级

- **Blocker**：破坏文件安全、证据可信性、隐私红线、回滚能力、主流程不可用
- **Major**：主要功能不完整、自动化用例失败、性能严重不达标
- **Minor**：边界显示问题、文案问题、个别非核心平台差异

### 1.3 验收执行方式

建议分三层执行：

1. **自动化层**：通过 `scripts/run_acceptance.py` 执行可自动断言的测试。
2. **人工冒烟层**：对 UI、引用面板、预览确认、风险提示做可视化复核。
3. **性能与打包层**：对性能阈值、发布目录、首次启动引导进行最终核验。

## 2. 环境与数据准备

### 2.1 最低环境

- Python 3.11
- 支持 SQLite FTS5
- 可用的本地文件系统沙箱目录
- 若要覆盖云端路径，需要至少一个可配置 Provider；若无密钥，须用 Mock Provider 完成自动化回归
- 用户时区与语言不影响本文件判定；报告建议用 `zh-CN`

### 2.2 测试目录建议

```text
.tmp/
├─ appdata/
├─ corpus/
├─ output/
├─ reports/
└─ sandbox_fs/
```

### 2.3 语料准备契约

必须通过以下脚本生成或加载验收语料：

```bash
python scripts/generate_fixture_corpus.py --profile acceptance --output .tmp/corpus
```

生成结果至少应包含：

- 项目周报 80 篇
- 会议纪要 80 篇
- 月报 / 季报 40 篇
- 研究笔记 40 篇
- 合同 / 报价 / 交付清单 20 篇
- 冲突版本文档 20 组
- 破损 / 空白 / 极短文档 20 个
- 黄金问题集 >= 100 条
- 归档规划场景样本 >= 20 组

### 2.4 报告输出契约

`run_acceptance.py` 至少输出：

- `report.json`
- `report.md`

建议 JSON 结构：

```json
{
  "suite": "full",
  "started_at": "2026-03-02T00:00:00Z",
  "finished_at": "2026-03-02T00:20:00Z",
  "summary": {
    "passed": 20,
    "failed": 0,
    "blocked": 0,
    "risk_count": 1
  },
  "cases": [
    {
      "id": "TC-001",
      "status": "passed",
      "duration_sec": 4.2,
      "severity": "Blocker",
      "artifacts": ["logs/audit.jsonl", "reports/tc-001.json"]
    }
  ],
  "metrics": {
    "search_p95_sec": 1.7,
    "citation_open_p95_sec": 0.8,
    "cold_start_sec": 5.6
  }
}
```

建议 Markdown 报告包含：

- 执行环境
- 版本信息
- 用例逐项结果
- 风险项
- 未完成项
- 关键性能指标
- 最终结论

## 4. 详细验收用例

### TC-001 新增根目录并扫描

- **级别**：Blocker
- **覆盖需求**：FR-001
- **类型**：Auto
- **前置条件**：
  - `.tmp/corpus/main` 已存在
  - 应用尚未配置该根目录
- **输入数据**：
  - 根目录路径
  - 排除规则：忽略隐藏文件、忽略 `.tmp`、忽略超大文件
  - 递归：开启
- **步骤**：
  1. 调用 `SourceService.add_source(...)`
  2. 调用 `SourceService.scan_source(...)`
  3. 读取 scan_run 结果与审计日志
- **预期结果**：
  - 根目录配置可保存并再次加载
  - 输出纳入 / 排除 / 失败统计
  - 生成 scan_run 记录
  - 生成审计记录
- **自动化断言建议**：
  - 断言 source_root 持久化存在
  - 断言扫描统计与语料实际数量一致
  - 断言 `audit.jsonl` 中存在 `scan_source` 事件
- **需保留证据**：
  - source 配置快照
  - scan_run 结果
  - 审计日志片段
- **失败判定**：
  - 不能保存根目录
  - 扫描统计错误
  - 无审计记录

### TC-002 不支持文件与单文件失败不拖垮批任务

- **级别**：Blocker
- **覆盖需求**：FR-001, FR-002
- **类型**：Auto
- **前置条件**：
  - 语料中存在不支持格式、损坏文档、无权限样本
- **输入数据**：
  - 包含 `.doc`、破损 `.pdf`、空文件、不可读文件的目录
- **步骤**：
  1. 发起扫描与索引
  2. 记录失败文件列表
  3. 检查其余有效文件是否完成入库
- **预期结果**：
  - 不支持文件进入排除清单
  - 失败文件被标记 `failed` 或 `partial`
  - 其余文件照常完成索引
- **自动化断言建议**：
  - 断言总任务未中断
  - 断言失败数 > 0 且成功数 > 0
  - 断言 Document 表中存在成功、失败两类状态
- **需保留证据**：
  - 失败清单
  - 总任务结果
- **失败判定**：
  - 任一单文件异常导致批任务整体失败
  - 失败未被记录

### TC-003 新增文件后的增量索引

- **级别**：Blocker
- **覆盖需求**：FR-002
- **类型**：Auto
- **前置条件**：
  - 已完成首次全量索引
- **输入数据**：
  - 向已监控目录新增一份合法 `.md` 或 `.docx` 文档
- **步骤**：
  1. 写入新文件
  2. 等待 watcher 或主动触发增量索引
  3. 搜索新文件中的唯一关键词
- **预期结果**：
  - 新文档写入 Document / Chunk / FTS / HNSW
  - 唯一关键词可被命中
  - 产生增量索引审计
- **自动化断言建议**：
  - 断言 `indexed_at` 晚于新增时间
  - 断言搜索结果包含新文档路径
- **需保留证据**：
  - 新文档样本
  - 搜索命中结果
- **失败判定**：
  - 新文件无法被搜索到
  - 监听存在明显延迟且无补偿机制

### TC-004 删除文件后的索引同步

- **级别**：Blocker
- **覆盖需求**：FR-002
- **类型**：Auto
- **前置条件**：
  - 已存在可检索文档
- **输入数据**：
  - 删除一个带唯一关键词的已索引文档
- **步骤**：
  1. 先搜索唯一关键词，确认命中
  2. 删除文件
  3. 触发 watcher 或手动增量更新
  4. 再次搜索唯一关键词
- **预期结果**：
  - Document 被标记 `is_deleted_from_fs = true` 或从活跃结果移除
  - 检索结果中不再出现该文档
- **自动化断言建议**：
  - 断言搜索结果为 0
  - 断言审计日志有 delete-from-fs 同步事件（不是系统 delete 功能）
- **需保留证据**：
  - 删除前后搜索截图或日志
- **失败判定**：
  - 已删除文件仍在结果中可见

### TC-005 混合搜索返回可引用结果

- **级别**：Blocker
- **覆盖需求**：FR-003, FR-015
- **类型**：Hybrid
- **前置条件**：
  - 完成全量索引
  - 黄金查询集已生成
- **输入数据**：
  - 5 条定位型查询
  - 5 条同义表达查询
  - 过滤条件组合 3 组
- **步骤**：
  1. 使用 SearchService 执行查询
  2. 检查 Top10 结果
  3. 人工点击至少 2 个引用跳转
- **预期结果**：
  - 每个结果含标题、路径、摘要、时间、分数、引用定位
  - Top10 命中率满足项目设定阈值
  - 引用能打开文件或定位到片段附近
- **自动化断言建议**：
  - 统计 Top10 命中率
  - 断言结果引用字段非空
- **需保留证据**：
  - 查询与结果日志
  - 引用点击录像或截图
- **失败判定**：
  - 返回结果无引用
  - 过滤器失效
  - 无法打开源文件/目录

### TC-006 事实问答默认附引用

- **级别**：Blocker
- **覆盖需求**：FR-004
- **类型**：Auto
- **前置条件**：
  - 至少一组问题具有确定答案且证据明确
- **输入数据**：
  - 单文档事实问答 3 条
  - 跨文档事实问答 3 条
- **步骤**：
  1. 调用 `QAService.answer(...)`
  2. 检查答案结构、引用列表、trace_id
- **预期结果**：
  - 所有事实性结论默认带引用
  - 引用指向真实存在的 chunk
  - 回答模板符合主规范 A 类样式
- **自动化断言建议**：
  - 断言 `answer.citations` 非空
  - 断言关键事实均被 citation validator 接受
- **需保留证据**：
  - 回答 JSON
  - 引用校验结果
- **失败判定**：
  - 有事实结论但无引用
  - 引用无法解析到 chunk

### TC-007 证据不足拒答

- **级别**：Blocker
- **覆盖需求**：FR-004
- **类型**：Auto
- **前置条件**：
  - 黄金问题集中存在故意无解或证据不足问题
- **输入数据**：
  - 证据不足问题至少 5 条
- **步骤**：
  1. 调用 `QAService.answer(...)`
  2. 检查是否输出拒答模板
- **预期结果**：
  - 返回“不足以可靠回答”或等价拒答
  - 给出已检查来源与建议下一步
  - 不伪造具体事实
- **自动化断言建议**：
  - 断言状态码或结果类型为 `insufficient_evidence`
  - 断言无未被证据支持的具体结论
- **需保留证据**：
  - 拒答结果
  - 检索到的候选证据摘要
- **失败判定**：
  - 明显胡答
  - 以记忆或猜测替代证据

### TC-008 冲突证据展示

- **级别**：Blocker
- **覆盖需求**：FR-004, FR-009
- **类型**：Auto
- **前置条件**：
  - 语料中至少存在 2 个版本冲突问题
- **输入数据**：
  - 冲突问题 3 条
- **步骤**：
  1. 执行问答
  2. 检查系统是否识别冲突
  3. 读取冲突来源列表
- **预期结果**：
  - 输出冲突说明而非单一确定结论
  - 至少展示 2 个冲突来源
  - 若有记忆冲突，标明“记忆可能陈旧或错误”
- **自动化断言建议**：
  - 断言 `result_type == conflict`
  - 断言 `conflict_sources >= 2`
- **需保留证据**：
  - 冲突回答
  - 来源 A/B 摘要
- **失败判定**：
  - 冲突场景被硬性归并成单一结论
  - 未展示来源

### TC-009 多文档决策摘要与来源追溯

- **级别**：Major
- **覆盖需求**：FR-005
- **类型**：Hybrid
- **前置条件**：
  - 有同一项目多份会议纪要 / 周报 / 月报
- **输入数据**：
  - 至少 8 篇同项目文档
- **步骤**：
  1. 选择多文档执行 summarize / extract_insights
  2. 查看决策、风险、待办列表
  3. 导出 Markdown
- **预期结果**：
  - 每个决策/风险/待办至少带一个来源引用
  - Markdown 导出成功
  - 人工抽查 3 条洞察，均能回到源文档
- **自动化断言建议**：
  - 断言导出的 Markdown 文件存在
  - 断言每一类洞察至少 1 项且带引用
- **需保留证据**：
  - 导出 Markdown
  - 洞察结果截图
- **失败判定**：
  - 洞察无来源
  - 导出失败

### TC-010 归档预览不执行

- **级别**：Blocker
- **覆盖需求**：FR-006, FR-011
- **类型**：Hybrid
- **前置条件**：
  - 至少 10 个待分类文档
- **输入数据**：
  - 待归档文件集
- **步骤**：
  1. 执行分类与归档规划
  2. 查看预览清单、目标路径、风险提示
  3. 在确认前中止
- **预期结果**：
  - 自动分类结果含置信度
  - 出现拟改动清单与目标路径
  - 用户取消后文件系统无任何变化
- **自动化断言建议**：
  - 断言生成 `FileOperationPlan(status=draft)`
  - 断言取消后目标路径为空或文件不存在变动
- **需保留证据**：
  - 预览清单
  - 取消后的目录快照
- **失败判定**：
  - 预览阶段即改动文件
  - 无风险提示或无目标路径清单

### TC-011 归档执行后可回滚

- **级别**：Blocker
- **覆盖需求**：FR-006, FR-011
- **类型**：Hybrid
- **前置条件**：
  - 已有通过预览的归档计划
- **输入数据**：
  - 包含 move / rename 的批量计划
- **步骤**：
  1. 用户确认执行
  2. 检查计划状态、目录变化、审计
  3. 调用“撤销最近一次批量操作”
  4. 校验路径、文件名、哈希恢复
- **预期结果**：
  - 执行前有确认动作
  - 执行后形成审计链路
  - 回滚后文件路径与名称恢复
  - 若部分失败，返回未恢复清单
- **自动化断言建议**：
  - 对执行前后做目录 diff
  - 断言 rollback 之后哈希与路径一致
- **需保留证据**：
  - 执行计划
  - 审计日志
  - 回滚结果报告
- **失败判定**：
  - 无法回滚
  - 回滚后文件不一致
  - 无审计

### TC-012 基于历史文档生成月报草稿

- **级别**：Major
- **覆盖需求**：FR-007
- **类型**：Hybrid
- **前置条件**：
  - 已有最近四周周报、会议纪要、风险清单
- **输入数据**：
  - 模板：月报
  - 时间范围：最近一个月
- **步骤**：
  1. 选择模板并生成草稿
  2. 检查引用区块
  3. 编辑 1 处内容
  4. 确认保存到输出目录
- **预期结果**：
  - 草稿可生成
  - 保存前可编辑
  - 保存前必须确认
  - 输出文件包含引用区块或尾注
- **自动化断言建议**：
  - 断言生成文件存在于 `OpenDocs_Output`
  - 断言内容包含引用标记
- **需保留证据**：
  - 输出文件
  - 保存审计日志
- **失败判定**：
  - 直接保存而无确认
  - 草稿无引用

### TC-013 M1 任务记忆 TTL

- **级别**：Major
- **覆盖需求**：FR-008
- **类型**：Auto
- **前置条件**：
  - M1 功能启用
- **输入数据**：
  - 一条任务记忆，TTL 设为 1 天（测试环境可缩短）
- **步骤**：
  1. 写入 M1
  2. 在 TTL 内搜索，确认能召回
  3. 通过时间旅行/模拟时间推进到过期后
  4. 再次搜索
- **预期结果**：
  - TTL 内可召回
  - 过期后被降权或标记 expired
  - 过期后默认不再参与正常召回
- **自动化断言建议**：
  - 断言状态从 active 变为 expired
- **需保留证据**：
  - 记忆状态前后对比
- **失败判定**：
  - 过期后仍正常参与召回
  - 无审计记录

### TC-014 删除或修正冲突记忆

- **级别**：Major
- **覆盖需求**：FR-008, FR-009
- **类型**：Hybrid
- **前置条件**：
  - 已存在与文档证据冲突的 M1 或 M2
- **输入数据**：
  - 一条冲突记忆
  - 对应真实文档证据
- **步骤**：
  1. 执行问答触发冲突提示
  2. 在记忆管理界面删除或修正该记忆
  3. 再次执行问答
- **预期结果**：
  - 冲突时明确提示
  - 删除/修正后不可再按旧内容召回
  - 再次问答时以文档证据为准
- **自动化断言建议**：
  - 断言冲突消失
  - 断言旧记忆状态为 disabled 或已更新
- **需保留证据**：
  - 修正前后结果
